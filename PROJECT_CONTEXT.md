# PROJECT_CONTEXT.md — 产品背景与核心需求

## 产品定位

个人自用的记账/资产记录 PWA 应用。**不做复杂流水记账**，核心目标是：记录各账户的当前余额，自动形成资产变化快照，展示总资产、净资产、负债及变化趋势。

## 使用场景

- 主要在 iPhone 上通过 Safari 访问
- 添加到主屏幕后像原生 App 一样使用
- 单人使用，无多用户、无协作、无账号体系

## 核心数据流

```
用户更新某账户余额 → 记录 BalanceHistory → 触发 AssetSnapshot 更新 → 首页汇总刷新
```

## 页面规划

只有 **一个页面**（首页），所有功能通过底部弹窗完成：

| 弹窗 | 触发方式 | 功能 |
|------|---------|------|
| 账户类型选择 | 首页点击加号 | 7 种简化类型可选 |
| 添加账户 | 选完类型后 | 填写名称、余额、备注 |
| 快速调整余额 | 点击账户行 | 输入新余额 |
| 编辑账户 | 账户行左滑→编辑 | 修改名称、类型、分组，二次确认后软删除 |
| 趋势图 | 顶栏趋势按钮 | 净资产趋势 + 总资产/总负债趋势 |
| 设置 | 顶栏齿轮按钮 | 数据导入导出、显示设置 |

## 交互原则

1. **点击加号 → 先选类型 → 再填信息**（两步，不是一步）
2. **所有弹窗高度为 80vh**，支持下滑关闭
3. **弹窗内容过多时弹窗内部滚动**，背景页面不动
4. **禁止双击缩放**（`touch-action: manipulation`）
5. **键盘弹起时页面布局尽量稳定**
6. **重要操作有反馈**（保存成功、删除成功、导出成功 Toast）
7. **危险操作必须二次确认**（删除账户、清空数据、导入覆盖）

## 数据安全原则

1. 不能随意破坏已有数据结构
2. 修改数据结构必须提供兼容旧数据的迁移方案
3. 删除账户是软删除，历史记录保留
4. 所有数据存在本机 IndexedDB，不上传云端
5. 支持 JSON 格式完整数据导出/导入

## 非目标（当前阶段不做）

- 复杂流水记账 / 收支分类
- 多人协作 / 家庭共享
- 后端账号体系 / 云同步
- 银行自动同步
- 预算管理
- 账单提醒
- 多币种支持（仅人民币 CNY）
